# selfkept

A small collection of self-contained, offline-first HTML tools for encrypted field logging and GPS navigation. No server, no accounts, no external dependencies — just single `.htm` files you can carry, email, or host anywhere.

---

## Tools

### `secure-log.htm` — Encrypted Geo-Journal

A portable encrypted location journal. Each entry stores a timestamp, GPS coordinates, accuracy, and a description. All data is encrypted inside the file itself using your passphrase, so the file can be freely copied or sent over insecure channels and is useless without the correct passphrase.

Key features:
- Passphrase-protected AES-256-GCM encryption with PBKDF2 key derivation
- GPS coordinates auto-filled from device location (manual entry also supported)
- Live GPS tracker with distance and bearing navigation to any stored record
- Self-cloning — download an encrypted copy of the file with your data embedded in it
- Version numbering on every exported clone
- Can also export a fresh blank clone to start a new journal with a new passphrase

### `secure-notes.htm` — Encrypted Notes Journal

The same concept as the geo-journal but simpler — a single free-text field per entry with a timestamp. Useful for any sensitive notes you want to carry in an encrypted self-contained file.

### `geolocator.htm` — Standalone GPS Navigator

A lightweight GPS tool with no encryption or data storage. Shows your live position (latitude, longitude, accuracy, altitude) and lets you enter a target coordinate to navigate toward, displaying real-time distance and bearing with a rotating compass.

Suitable for hosting on a public website as it contains no personal data.

## How the self-contained format works

Each file is a complete, standalone HTML page. When you add records and download a clone, the tool re-generates the entire HTML file with your encrypted data embedded directly in the source as a base64 string. The next person to open that file sees only a passphrase prompt — without the correct passphrase the data cannot be decrypted.

This makes the files safe to send by email, store in cloud drives, or share via any channel regardless of whether that channel is secure.

## Device and browser compatibility

These tools run entirely in the browser with no installation required. However, two browser capabilities are needed depending on the tool:

**Web Crypto API** — required by `secure-log.htm` and `secure-notes.htm` for encryption and decryption. Supported in all modern desktop browsers (Chrome, Firefox, Edge, Safari) and on Android. On iOS, the Web Crypto API is blocked when opening files directly from the Files app (`file://` URLs) — to use the encrypted journals on iPhone they should be served from a web server over HTTPS.

**Geolocation API** — required by `secure-log.htm` and `geolocator.htm` for GPS functions. The browser will prompt for location permission on first use. On mobile devices, geolocation requires the page to be served over HTTPS — it will not work on `file://` URLs on iOS or modern Android. Manual coordinate entry always works regardless.

**`geolocator.htm`** is the simplest to host — just place it on any HTTPS website and it works on all devices including iPhone.

**In summary:** for desktop use, all three files work standalone straight from your file system. For iPhone use, hosting on a simple HTTPS web server is recommended.

## Security notes

- Encryption uses AES-256-GCM with a random salt and IV per export, and PBKDF2 with 310,000 iterations for key derivation
- The encrypted payload is the only secret — the encryption algorithm is visible in the file source, which is intentional (security through obscurity is not relied upon)
- Security is entirely dependent on passphrase strength — use a strong, unique passphrase
- No data is ever transmitted anywhere — everything happens locally in your browser
